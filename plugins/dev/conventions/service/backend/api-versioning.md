# API 버저닝 & Deprecation

> 버전: v1.0.0
> 최종 수정: 2026-06-12
> 적용 조건: 다음 중 하나라도 해당하면 적용
> - 외부/공개에 노출되어 클라이언트가 독립적으로 갱신되는 HTTP API인 경우
> - 하위호환을 깨는 변경이 발생할 수 있고, 구버전과 신버전을 동시 운용해야 하는 경우
>
> socket·gRPC 등 비 HTTP 통신은 범위 밖이다.

---

## 메타-규칙

API 버전을 **어떻게 노출하고, 어떻게 폐기(deprecate)하는지를 전역 단일 정책**으로 정한다. 버전 식별자를 어디에 둘지(URL prefix vs 헤더)는 `api-path.md`의 prefix 결정과 **한 곳에서 정해 충돌을 막는다** — 본 컨벤션은 *버저닝 전략과 deprecation 절차*를 정하고, prefix 문자열 자체(`/api/v1`)는 `api-path.md`가 정한다. 버전 협상을 `Accept` 헤더로 둘지의 결정은 `content-negotiation.md`와 정합해야 한다.

> **검증 경로**: 버저닝/deprecation 정책이 상세설계에 박혔는지는 Stage 2 LLM judge가, 폐기 버전 호출 응답 등 동작은 필요 시 fragment "검증 항목"으로 런타임 `verifier`가 확인한다.

## 결정 항목

이 컨벤션을 따르려면 다음 항목들을 결정하여 상세설계서에 명시해야 한다.

| 결정 항목 | 명시 위치 | 형태 / 제약 |
|----------|---------|-----------|
| 버전 노출 위치 | `service/{백엔드 서비스}.md` / 버저닝 섹션 | 셋 중 택일: ① URL path prefix(`/api/v1`) ② 헤더(`Accept` 미디어 타입 또는 커스텀) ③ 쿼리 파라미터. `api-path.md`의 prefix 결정과 일치해야 함 |
| 버전 증가 기준 | `service/{백엔드 서비스}.md` / 버저닝 섹션 | 무엇이 버전(major)을 올리는가 = 하위호환을 깨는 변경의 정의(필드 제거·의미 변경·필수화 등). 비파괴 변경은 버전 유지 |
| 비파괴 변경 정의 | `service/{백엔드 서비스}.md` / 버저닝 섹션 | 같은 버전 내에서 허용되는 변경(필드 추가 등)과 클라이언트에 기대하는 수용성(tolerant reader: 모르는 필드 무시) 명시 |
| 버전 적용 범위 | `service/{백엔드 서비스}.md` / 버저닝 섹션 | 서비스 전역 단일 버전 vs 도메인별 버전. 보통 서비스 전역 단일 권장 |
| 동시 지원 버전 수 | `service/{백엔드 서비스}.md` / 버저닝 섹션 | 동시에 운용하는 버전 개수 정책(예: 최신 + 직전 1개) |
| Deprecation 절차 | `service/{백엔드 서비스}.md` / 버저닝 섹션 | 폐기 예고 방법(`Deprecation`/`Sunset` 헤더, 문서 공지)·예고 기간·제거 시점 결정 |
| 폐기 후 호출 응답 | `service/{백엔드 서비스}.md` / 버저닝 섹션 | 제거된 버전 호출 시: `410 Gone` vs 최신으로 안내. 코드는 `error-catalog.md` |

## 상세설계 작성 예

`service/{백엔드 서비스}.md` 에 다음 섹션을 추가한다.

```markdown
## 버저닝 & Deprecation

| 항목 | 정책 |
|------|------|
| 버전 노출 | URL path prefix `/api/v{N}` (`api-path.md`와 동일). 헤더 협상 미사용 |
| 버전 증가 기준 | 하위호환 파괴 시에만 major 증가: 필드 제거·타입 변경·필수화·의미 변경 |
| 비파괴 변경 | 같은 버전 내 허용: optional 필드 추가, enum 값 추가. 클라이언트는 모르는 필드 무시(tolerant reader) |
| 적용 범위 | 서비스 전역 단일 버전 |
| 동시 지원 | 최신 + 직전 1개 버전 |
| Deprecation | `Deprecation: true` + `Sunset: <date>` 헤더로 예고. 최소 90일 유예 후 제거 |
| 폐기 후 호출 | 제거된 버전 → `410 GONE` (최신 버전 경로 안내 메시지 포함) |
```
