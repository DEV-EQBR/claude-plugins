# 목록 응답 구조

> 버전: v1.0.0
> 최종 수정: 2026-05-19
> 적용 조건: 다음 중 하나라도 해당하면 적용
> - 도메인이 컬렉션을 반환하는 GET endpoint를 제공하는 경우 (`/users`, `/projects` 등)
> - 페이지네이션·검색·정렬 파라미터가 필요한 endpoint가 있는 경우

---

## 메타-규칙

목록 응답은 프로젝트 전역에서 **단일 구조**를 따른다. 배열 키 명명, 페이지네이션 키 명명, 검색·정렬 쿼리 파라미터까지 도메인 간 갈리지 않게 통일한다. 도메인 api.md는 목록 응답 구조를 자체 결정하지 않고 본 컨벤션이 박힌 결정을 따른다. 외부 API(GitHub, AWS SDK 등) 응답을 그대로 노출하는 endpoint는 본 컨벤션의 적용 예외로 명시할 수 있다. 목록 각 항목(리소스)의 필드 표현·요약 vs 상세 구분은 `resource-representation.md`에서 결정한다.

---

## 결정 항목

이 컨벤션을 따르려면 다음 항목들을 결정하여 상세설계서에 명시해야 한다.

| 결정 항목 | 명시 위치 | 형태 / 제약 |
|----------|---------|-----------|
| 배열 키 명명 | `service/{백엔드 서비스}.md` / 목록 응답 컨벤션 섹션 | 단일 키 강제 (보통 `items`). 도메인명을 키로 사용(`users`, `projects`) 금지 |
| 페이지네이션 객체 키 명명 | `service/{백엔드 서비스}.md` / 목록 응답 컨벤션 섹션 | 객체 명칭 단일화 (보통 `pagination`). 내부 키 상세설계: `page` / `limit`(또는 `size` 택1) / `total`(또는 `totalElements` 택1) / `totalPages`(또는 `total_pages` — `field-naming.md`의 케이싱 정책과 일치) |
| 페이지네이션 적용 범위 | `service/{백엔드 서비스}.md` / 목록 응답 컨벤션 섹션 | 모든 목록에 강제 vs 일부 예외 허용. 예외 허용 시 예외 조건(소규모 고정 목록, 외부 API 그대로 노출 등) 명시 |
| 검색 쿼리 파라미터 | `service/{백엔드 서비스}.md` / 목록 응답 컨벤션 섹션 | 표준 파라미터명 결정 (보통 `search` 또는 `q`). 빈 값/미전송 동작(필터 미적용)도 명시. 도메인별 검색 매칭 필드는 도메인 api.md에서 정의 |
| 정렬 쿼리 파라미터 | `service/{백엔드 서비스}.md` / 목록 응답 컨벤션 섹션 | 셋 중 택일: ① `sortBy` + `sortOrder` 분리 ② `sort=field:asc` 단일 ③ 정렬 미지원 (도메인별 고정 정렬). 정렬 가능 필드 화이트리스트 결정 방식도 명시 |
| 페이지 파라미터 기본값·상한 | `service/{백엔드 서비스}.md` / 목록 응답 컨벤션 섹션 | `page` 기본값(보통 `1`, 1-base) / `limit` 기본값(보통 `20`)과 최대값(보통 `100`) 명시 |
| 페이지네이션 외 모드 | `service/{백엔드 서비스}.md` / 목록 응답 컨벤션 섹션 | 무한 스크롤(cursor 기반), 외부 API token 기반(NextToken), 캘린더 모드(전체 조회) 등 별도 모드 사용 시 트리거 조건과 응답 차이 명시 |

## 상세설계 작성 예

`service/{백엔드 서비스}.md` 에 다음 섹션을 추가한다.

```markdown
## 목록 응답 컨벤션

모든 목록 endpoint는 아래 단일 구조를 따른다.

\`\`\`json
{
  "success": true,
  "data": {
    "items": [ ... ],
    "pagination": {
      "page": 1,
      "limit": 20,
      "total": 153,
      "totalPages": 8
    }
  }
}
\`\`\`

### 쿼리 파라미터

| 파라미터 | 타입 | 기본값 | 제약 | 설명 |
|---------|------|--------|------|------|
| `search` | string | (없음) | optional | 검색어. 빈 문자열·미전송이면 필터 미적용 |
| `page` | number | 1 | 1-base, min 1 | 페이지 번호 |
| `limit` | number | 20 | max 100 | 페이지당 항목 수 |
| `sortBy` | string | (도메인별 기본값) | 화이트리스트 검증 | 정렬 기준 필드명 |
| `sortOrder` | enum | `asc` | `asc` / `desc` | 정렬 방향 |

### 페이지네이션 적용 예외

- 외부 API 응답을 그대로 노출하는 endpoint: 본 컨벤션 미적용 (각 도메인 api.md에 명시)
- 캘린더 모드(전체 조회): `page`/`limit` 미전달 시 트리거. 페이지네이션 객체는 동일 구조로 채우되 `limit=total`, `totalPages=1`
```
