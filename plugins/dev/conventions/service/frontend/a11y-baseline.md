# 접근성 베이스라인 (a11y)

> 버전: v1.0.0
> 최종 수정: 2026-06-26
> 적용 조건: 다음에 해당하면 적용
> - 화면(`screens.md`)이 정의된 모든 프론트엔드 서비스 (플랫폼 무관 — web·네이티브 공통)

---

## 메타-규칙

화면은 마우스를 쓰지 않거나 화면을 또렷이 보지 못하는 사용자도 **쓸 수 있어야** 한다. 접근성은 사후 보정이 아니라 베이스라인이다 — 시맨틱 역할·키보드 동작·충분한 대비·포커스 가시성·적절한 터치 타깃은 모든 화면이 기본으로 충족한다. 이 베이스라인은 품질의 바닥이자, 검증 가능한 신호다(대비·역할·라벨·타깃 크기는 자동 감사로 측정된다 — `verify`의 시각 게이트가 axe류로 검사). 접근성 있는 마크업은 UI 자동화 검증의 선택자(role·label)도 안정화한다.

> 이 컨벤션은 *최소 충족 기준*을 강제한다. 화면별 *구체 라벨 문구·포커스 순서*는 `ui.md`/`screens.md`에 설계된다.

## 결정 항목

이 컨벤션을 따르려면 다음 항목들을 결정하여 명세서에 명시해야 한다.

| 결정 항목 | 명시 위치 | 형태 / 제약 |
|----------|---------|-----------|
| 준수 기준 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 목표 기준(기본: WCAG 2.1 AA). 색 대비 본문 4.5:1 / 큰 글자·UI 요소 3:1 |
| 키보드/포커스 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 모든 인터랙티브 요소 키보드 도달·조작 가능, 가시 포커스 링 의무, 논리적 포커스 순서, 모달 포커스 트랩 |
| 시맨틱·라벨 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 의미에 맞는 역할/시맨틱 요소 사용(div 남용 금지), 아이콘 단독 버튼·입력에 접근성 라벨 의무 |
| 터치 타깃 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 최소 타깃 크기(예: 44×44pt 권장 / 최소 24×24 + 충분한 간격). 폼팩터별 기준은 `screen-layout.md` 참조 |

## spec 작성 예

`service/{프론트엔드 서비스}.md` 에 다음 섹션을 추가한다.

```markdown
## 디자인 정책 - 접근성 베이스라인

| 항목 | 정책 |
|------|------|
| 준수 기준 | WCAG 2.1 AA — 본문 대비 4.5:1, 큰 글자/UI 요소 3:1 (디자인 토큰 색 조합이 이를 만족하도록 선택) |
| 키보드/포커스 | 모든 인터랙티브 요소 Tab 도달·Enter/Space 조작, 가시 포커스 링 의무, 모달 포커스 트랩 + Esc 닫기 |
| 시맨틱·라벨 | 버튼/링크/헤딩/리스트 시맨틱 사용, 아이콘 단독 버튼에 `aria-label`, 입력에 연결된 label |
| 터치 타깃 | 터치 환경 최소 44×44pt, 인접 타깃 간 8px 이상 간격 |
```
