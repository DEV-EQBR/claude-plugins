# 날짜·시간 표시

> 버전: v1.0.0
> 최종 수정: 2026-05-04
> 적용 조건: 다음 중 하나라도 해당하면 적용
> - 화면(`screens.md`)에 날짜 또는 시간 표시 요소가 있는 경우
> - 화면에 그래프/차트가 있고 축에 날짜/시간이 사용되는 경우

---

## 메타-규칙

프로젝트는 **단일 표시 형식 정책**을 정하고, 모든 화면이 동일 형식을 따른다. 데이터의 정밀도(날짜만/분 단위/초 단위)나 표시 공간(좁은 차트 축 등)에 따라 정밀도와 길이의 변형만 허용한다. 표시 시간대는 `system/timezone.md` 의 결정을 따른다.

---

## 결정 항목

이 컨벤션을 따르려면 다음 항목들을 결정하여 명세서에 명시해야 한다.

| 결정 항목 | 명시 위치 | 형태 / 제약 |
|----------|---------|-----------|
| 기본 날짜 형식 | `service/{프론트엔드 서비스}.md` / 표시 형식 섹션 | 단일 포맷 문자열 (예: `yyyy.MM.dd`). 구분자도 함께 결정 |
| 분 단위 형식 | `service/{프론트엔드 서비스}.md` / 표시 형식 섹션 | 시간까지 표시하는 형식 (예: `yyyy.MM.dd HH:mm`) |
| 초 단위 형식 | `service/{프론트엔드 서비스}.md` / 표시 형식 섹션 | 초 단위까지 필요할 때의 형식 (예: `yyyy.MM.dd HH:mm:ss`). 사용 안 하면 "사용 안 함" 명시 |
| 좁은 공간용 단축 형식 | `service/{프론트엔드 서비스}.md` / 표시 형식 섹션 | 차트 축 등 좁은 공간 단축형 (예: `MM-DD`, `HH:mm`). 적용 조건도 함께 명시 |
| 정밀도 선택 기준 | `service/{프론트엔드 서비스}.md` / 표시 형식 섹션 | 어떤 데이터 유형에 어떤 정밀도를 쓰는지 매핑 (예: 거래 내역 = 분 단위, 회원 가입일 = 날짜만) |

---

## spec 작성 예

`service/{프론트엔드 서비스}.md` 에 다음 섹션을 추가한다.

```markdown
## 표시 형식 - 날짜·시간

| 정밀도 | 형식 | 사용 컨텍스트 |
|-------|------|------------|
| 날짜만 | `yyyy.MM.dd` | 가입일, 만료일 등 |
| 분 단위 | `yyyy.MM.dd HH:mm` | 거래 내역, 이벤트 로그 등 |
| 초 단위 | (사용 안 함) | — |
| 단축형 | `MM-DD` | 차트 X축 (1년 이내 데이터) |
| 단축형 (장기) | `yyyy-MM` | 차트 X축 (1년 초과 데이터) |

> 표시 시간대는 `system.md` 시간 데이터 처리 섹션의 정책을 따른다.
```
