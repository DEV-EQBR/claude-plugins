# 디자인 토큰 (색·간격·radius·elevation·타입 스케일)

> 버전: v1.0.0
> 최종 수정: 2026-06-26
> 적용 조건: 다음에 해당하면 적용
> - 화면(`screens.md`)이 정의된 모든 프론트엔드 서비스 (플랫폼 무관 — web·네이티브 공통)

---

## 메타-규칙

화면의 모든 시각 값(색·간격·모서리·그림자·글자 크기)은 **명명된 토큰의 닫힌 집합**에서만 나온다. 컴포넌트에 매직값(`#3b7af7`, `13px`, `7px`)을 직접 쓰지 않는다 — 토큰을 참조한다. 토큰은 단일 출처(디자인 정책)에서 정의되어 변경 시 일괄 반영된다. 색은 원시 색이 아니라 **의미 역할(semantic role)**로 정의한다 — 컴포넌트는 "파랑"이 아니라 "primary"를 참조하므로, 테마·다크모드·리브랜딩이 토큰 교체만으로 끝난다.

> 이 컨벤션은 토큰의 *구조*(어떤 역할·스케일이 있어야 하는가)를 강제한다. 토큰의 *구체 값*(어떤 파랑인가)은 프로젝트가 디자인 정책에 박는 결정이다.

이 컨벤션을 따르려면 다음 항목들을 결정하여 상세설계서에 명시해야 한다.

| 결정 항목 | 명시 위치 | 형태 / 제약 |
|----------|---------|-----------|
| 색 역할 팔레트 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 의미 역할별 색 토큰. 최소 역할: `primary`(주 액션) · `surface`(배경/카드) · `text`(본문/보조 2단계 이상) · `border`(구분선) · `success`/`warning`/`danger`(상태). 각 역할은 hover/active/disabled 변형 또는 그 도출 규칙을 가진다 |
| 다크모드 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 셋 중 택일: ① 미지원(라이트 고정) ② 라이트+다크 2테마 ③ 시스템 설정 추종. 지원 시 역할 토큰을 테마별로 정의 |
| 간격 스케일 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 단일 베이스 단위(예: 4px)의 배수로 된 닫힌 스케일(예: 4·8·12·16·24·32·48). 임의 간격 금지 |
| Radius 스케일 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 모서리 반경 토큰 집합(예: none·sm·md·lg·full). 컴포넌트 종류별 적용 기준 |
| Elevation(그림자) 스케일 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 깊이 단계 토큰(예: 0·1·2·3). 또는 "그림자 미사용(border로 깊이 표현)" 명시 |
| 타입 스케일 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 글자 크기·굵기·행간의 명명된 단계(예: display·title·body·caption). `typography.md`의 본문 폰트 패밀리 위에 얹는다 |

## 상세설계 작성 예

`service/{프론트엔드 서비스}.md` 에 다음 섹션을 추가한다.

```markdown
## 디자인 정책 - 디자인 토큰

### 색 (의미 역할)
| 역할 | 라이트 | 다크 | 비고 |
|------|--------|------|------|
| primary | `#2563EB` | `#3B82F6` | hover = 10% 어둡게, disabled = 40% 불투명 |
| surface | `#FFFFFF` / `#F8FAFC`(2단) | `#0F172A` / `#1E293B` | 배경·카드 |
| text | `#0F172A`(본문) / `#64748B`(보조) | `#F1F5F9` / `#94A3B8` | 2단 위계 |
| border | `#E2E8F0` | `#334155` | 구분선·입력 테두리 |
| success / warning / danger | `#16A34A` / `#D97706` / `#DC2626` | 동일 계열 | 상태 표현 |

### 다크모드
시스템 설정 추종 (라이트+다크 역할 토큰 모두 정의)

### 간격 스케일
베이스 4px. 토큰: `1`(4) · `2`(8) · `3`(12) · `4`(16) · `6`(24) · `8`(32) · `12`(48). 그 외 간격 금지.

### Radius
`none`(0) · `sm`(4) · `md`(8) · `lg`(16) · `full`(9999). 입력/버튼=md, 카드=lg, 아바타/칩=full.

### Elevation
`0`(없음·border로 구분) · `1`(카드 hover) · `2`(드롭다운/팝오버) · `3`(모달/토스트).

### 타입 스케일
| 토큰 | 크기/행간 | 굵기 | 용도 |
|------|-----------|------|------|
| display | 28/36 | 700 | 페이지 타이틀 |
| title | 18/26 | 600 | 섹션 제목 |
| body | 15/22 | 400 | 본문 |
| caption | 13/18 | 400 | 보조 텍스트 |
```
