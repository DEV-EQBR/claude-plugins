## 메타-규칙

모션은 장식이 아니라 **상태 변화를 사용자에게 설명하는 수단**이다 — 무엇이 나타나고/사라지고/이동했는지, 어떤 액션이 접수됐는지를 전달한다. 모션도 토큰화한다(duration·easing의 닫힌 집합) — 화면마다 제각각인 전이 속도는 산만하다. 모션은 절제한다: 빠르고(대개 150–300ms) 목적이 있어야 하며, 과한 애니메이션은 느리고 조잡하게 느껴진다. **접근성**: OS의 "동작 줄이기(reduced motion)" 설정을 존중하여 비필수 모션을 끈다.

모션 요구는 **검증 가능해야 한다**. "자연스럽게·부드럽게" 같은 형용사는 사람마다 다르게 읽혀 그대로는 상세설계가 될 수 없다 — 모션 요구가 상세설계에 들어올 때는 **관찰 가능한 불변식**(횟수·기하·시간)으로 번역해 기술한다(아래 "모션 불변식"). 감성 품질(easing의 체감)은 자동 판정 대상이 아니므로 **"사람 확인 항목"**으로 분리 표기한다.

> 이 컨벤션은 모션의 *토큰화·절제·접근성·검증 가능성 원칙*을 강제한다. 화면별 *어떤 요소에 어떤 전이를 거는가*는 `ui.md`에 설계된다.

## 모션 불변식 — 형용사를 번역하는 규칙

모션은 시간축 위의 속성이라, 정적 확인(소스 검사·단일 스크린샷)으로는 검증되지 않는다. 그래서 모션 요구는 아래 3형식의 **불변식**으로 기술한다 — 전부 실행 중 관찰 사실(횟수 세기·좌표 비교·시간 측정)로 판정 가능한 형태다.

| 형식 | 무엇을 고정하는가 | 예 |
|------|-----------------|-----|
| **횟수** | 트리거 1회당 전이(애니메이션) 실행 횟수 | "항목 1개 추가 시 등장 전이 정확히 1회 — 기존 항목 재-전이 0회" |
| **기하** | 정착 후 요소·영역의 좌표 관계 | "정착 후 신규 항목의 경계가 가시 영역 안에 있음 (잘림 0)" |
| **시간** | 트리거 후 상태 도달의 상한 | "추가 후 {duration 토큰 + 여유} 내 정착" |

### 번역 카탈로그

요구사항/설계에서 자주 오는 형용사 표현의 표준 번역. writer는 모션 요구를 만나면 이 카탈로그에서 해당 세트를 찾아 전개한다 — 형용사 뒤에 숨은 불변식을 백지에서 발명하면 누락이 생기므로, 카탈로그 대조를 기본으로 한다.

| 요구 표현 | 표준 불변식 세트 |
|----------|----------------|
| "리스트/피드/대화에 자연스럽게 추가" | 신규 항목만 등장 전이 1회 · 기존 항목 재-전이 0회(렌더 identity 안정) · 정착 후 신규 항목이 가시 영역 안(잘림 금지 — 필요 시 앵커 스크롤 동반) · duration 토큰 내 정착 |
| "부드러운 화면/뷰 전환" | 전환 전이 1회 · 전환 중 빈 상태(이전·다음 화면 모두 안 보임) 금지 · duration 토큰 내 완료 |
| "튀지 않는 로딩/콘텐츠 교체" | 교체 시 기존 가시 요소의 위치 이동(레이아웃 시프트) 금지(자리 선확보) · 교체 전이 1회 |
| "매끄러운 추종(드래그·스크롤 연동)" | 입력 대비 표시 지연 상한 · 입력 종료 후 최종 위치 정착(오버슈트 허용 여부 명시) |

카탈로그에 없는 표현은 같은 3형식으로 직접 번역하고, 반복 등장하면 이 카탈로그에 추가를 제안한다.

### 수치 규칙

- 불변식의 시간 상한은 이 컨벤션의 **duration 토큰에서 도출**한다 (예: 정착 상한 = 해당 전이의 토큰 + 여유분).
- 요구가 수치를 못박지 않아 표준값을 채웠으면 **가정으로 표기**한다 — 그럴듯한 기본값을 확인된 값처럼 적지 않는다.
- 어느 불변식 세트를 뜻하는지가 요구에서 갈리면(같은 형용사가 다른 속성을 가리킬 수 있음) 번역을 가정하지 말고 결정 요청으로 올린다 — 의도 오역이 모션 상세설계의 최대 실패 모드다.

## 결정 항목

이 컨벤션을 따르려면 다음 항목들을 결정하여 상세설계서에 명시해야 한다.

| 결정 항목 | 명시 위치 | 형태 / 제약 |
|----------|---------|-----------|
| Duration 토큰 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 전이 시간의 닫힌 집합(예: fast 150 · base 200 · slow 300ms). 용도별 매핑 |
| Easing 토큰 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 가속 곡선 집합(예: standard·decelerate·accelerate). 진입/이탈 기준 |
| 적용 범위 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 모션을 거는 표준 상황(상태 전이·등장/퇴장·피드백·페이지 전환). "거의 안 씀"도 명시 가능 |
| Reduced motion 대응 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | OS 동작 줄이기 설정 시 처리(비필수 모션 제거 / 페이드만 유지 등). 필수 — 미대응 금지 |
| 모션 불변식 표기 | `domain/{도메인}/ui.md` / 각 화면 모션 섹션 | 모션이 요구된 화면은 해당 모션의 불변식(횟수·기하·시간 — 위 3형식)을 명시. 형용사 단독 기술 금지. 자동 판정 불가한 감성 품질은 "사람 확인 항목"으로 분리 표기 |

## 상세설계 작성 예

`service/{프론트엔드 서비스}.md` 에 다음 섹션을 추가한다.

```markdown
## 디자인 정책 - 모션

| 항목 | 정책 |
|------|------|
| Duration | fast 150ms(상태 전이·hover) · base 200ms(드롭다운/토스트 등장) · slow 300ms(모달/시트) |
| Easing | standard `cubic-bezier(.2,0,0,1)` 기본 · decelerate(진입) · accelerate(이탈) |
| 적용 범위 | 상태 전이(hover/focus), 오버레이 등장/퇴장(페이드+슬라이드), 토스트, 스켈레톤→콘텐츠 크로스페이드. 장식성 애니메이션 없음 |
| Reduced motion | `prefers-reduced-motion: reduce` 시 이동/스케일 제거, 0–100ms 페이드만 유지 |
```

모션 불변식이 `domain/{도메인}/ui.md`의 화면 모션 섹션에 박히는 예 (요구: "대화창에 메시지가 메신저처럼 자연스럽게 추가").

```markdown
### 모션

| 전이 | 토큰 | 불변식 |
|------|------|--------|
| 대화 턴 추가 | base 200ms · decelerate | 횟수: 신규 턴만 등장 전이 1회, 기존 턴 재-전이 0회 (렌더 identity 안정) / 기하: 정착 후 신규 턴 하단 경계 ≤ 가시 영역 하단 (하단 앵커 스크롤 동반, 잘림 0) / 시간: 턴 추가 후 300ms 내 정착 (가정: base + 100ms 여유) |

사람 확인 항목: 등장 easing의 체감 (스프링 느낌)
```
