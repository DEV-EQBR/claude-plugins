# 수 표시

> 버전: v1.0.0
> 최종 수정: 2026-05-04
> 적용 조건: 다음 중 하나라도 해당하면 적용
> - 화면(`screens.md`)에 숫자(수량/통화/지표/카운트 등)를 표시하는 요소가 있는 경우
> - 화면에 그래프/차트가 있는 경우 (축·값 라벨에 숫자 표시)

---

## 메타-규칙

수 표시는 프로젝트의 **표시 locale** 형식 규칙(그룹 구분자, 소수점 등)을 따른다. 단위가 컨텍스트로 명확하지 않은 수에는 단위를 함께 표기하며, 단위 표기 위치와 형식은 일관되게 적용한다. 큰 수의 축약 표시 정책은 프로젝트가 명시적으로 정한다.

---

## 결정 항목

이 컨벤션을 따르려면 다음 항목들을 결정하여 상세설계서에 명시해야 한다.

| 결정 항목 | 명시 위치 | 형태 / 제약 |
|----------|---------|-----------|
| 표시 locale | `service/{프론트엔드 서비스}.md` / 표시 형식 섹션 | BCP 47 locale 코드 (예: `ko-KR`, `en-US`). 다국어 지원 시 locale 결정 방식도 함께 명시 |
| 큰 수 축약 정책 | `service/{프론트엔드 서비스}.md` / 표시 형식 섹션 | 둘 중 택일: ① 축약 안 함 (항상 풀 표기) ② 임계값 이상부터 축약 + 단위 어휘(예: 만/억/조 또는 K/M/B) |
| 단위 표기가 필요한 데이터 유형 | `service/{프론트엔드 서비스}.md` / 표시 형식 섹션 | 단위 표기가 필수인 데이터 유형 목록 (예: 통화, 비율, 카운트, 시간/거리 측정값 등) |
| 단위 표기 위치·형식 | `service/{프론트엔드 서비스}.md` / 표시 형식 섹션 | 데이터 유형별 단위 표기 위치(제목 우측/인라인/축 라벨 등)와 형식 (예: `(단위:KRW)`, `5%`, `1,234건`) |

---

## 상세설계 작성 예

`service/{프론트엔드 서비스}.md` 에 다음 섹션을 추가한다.

```markdown
## 표시 형식 - 수

| 항목 | 정책 |
|------|------|
| 표시 locale | `ko-KR` (단일 locale) |
| 그룹 구분자 / 소수점 | locale 기본값 (`,` / `.`) |
| 큰 수 축약 | 1만 이상부터 축약. 단위 어휘: 만/억/조 |

### 단위 표기

| 데이터 유형 | 표기 위치 | 표기 형식 |
|-----------|---------|---------|
| 통화 (KRW 등) | 그래프/표 제목 우측 | `(단위:KRW)` |
| 비율 (%) | 인라인 (값 우측) | `5%` |
| 카운트 (건수) | 그래프 제목 우측 | `(단위:건)` |
```
