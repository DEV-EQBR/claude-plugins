# 타이포그래피

> 버전: v1.0.0
> 최종 수정: 2026-05-04
> 적용 조건: 다음 중 하나라도 해당하면 적용
> - 화면(`screens.md`)이 정의된 모든 프론트엔드 서비스
> - 사용자에게 텍스트가 표시되는 모든 화면

---

## 메타-규칙

한 프로젝트는 **단일 본문 폰트 패밀리**만 사용한다. 본문에 코드/숫자 정렬이 본질적으로 필요하지 않다면 모노스페이스 폰트는 혼용하지 않는다. 폰트는 디자인 토큰으로 관리하여 변경 시 일괄 적용되도록 한다.

> 이 컨벤션은 폰트 *패밀리*를 정한다. 글자 크기·굵기·행간의 명명된 단계(타입 스케일)는 `design-tokens.md`가 정의하며, 그 스케일이 여기서 정한 본문 폰트 패밀리 위에 얹힌다.

---

## 결정 항목

이 컨벤션을 따르려면 다음 항목들을 결정하여 상세설계서에 명시해야 한다.

| 결정 항목 | 명시 위치 | 형태 / 제약 |
|----------|---------|-----------|
| 본문 폰트 패밀리 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 단일 폰트 패밀리 이름 (예: `Noto Sans KR`, `Inter` 등). 로딩 방식(웹폰트/시스템 폰트)도 함께 명시 |
| Fallback 체인 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 본문 폰트 미지원 시 fallback 순서. 시스템 sans-serif로 종료 |
| 모노스페이스 사용 영역 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 셋 중 택일: ① 사용 안 함 ② 코드 블록 한정 ③ 명시한 영역 한정 (영역 목록 포함) |

---

## 상세설계 작성 예

`service/{프론트엔드 서비스}.md` 에 다음 섹션을 추가한다.

```markdown
## 디자인 정책 - 타이포그래피

| 항목 | 정책 |
|------|------|
| 본문 폰트 | `Noto Sans KR` (Google Fonts 웹폰트 로딩) |
| Fallback | `Apple SD Gothic Neo`, `Malgun Gothic`, `sans-serif` |
| 모노스페이스 사용 | 사용 안 함 (본문에 코드/숫자 정렬 목적 없음) |
```
