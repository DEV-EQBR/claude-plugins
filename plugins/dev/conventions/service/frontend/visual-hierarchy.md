# 시각 위계 (밀도·정렬·강조)

> 버전: v1.0.0
> 최종 수정: 2026-06-26
> 적용 조건: 다음에 해당하면 적용
> - 화면(`screens.md`)이 정의된 모든 프론트엔드 서비스 (플랫폼 무관 — web·네이티브 공통)

---

## 메타-규칙

한 화면 안의 요소들은 **중요도 순서가 시각적으로 읽혀야** 한다. 사용자가 "여기서 가장 중요한 것/해야 할 행동"을 0.5초 안에 찾을 수 있어야 한다. 위계는 크기·굵기·색·간격·정렬로 만들어진다(`design-tokens.md`의 토큰 위에서). 모든 것을 똑같이 강조하면 아무것도 강조되지 않는다 — 1차 행동은 하나만 강조하고 나머지는 낮춘다. 간격은 관계를 표현한다: 가까운 것은 묶이고 먼 것은 분리된다(근접성). 정렬은 단일 그리드를 따른다 — 들쭉날쭉한 가장자리는 조잡함으로 읽힌다.

> 이 컨벤션은 위계를 만드는 *규칙*을 강제한다. 화면별 *실제 배치/강조 대상*은 `ui.md`에 설계된다. 반응형 브레이크포인트·폼팩터는 `screen-layout.md`가 정한다(여기서 중복하지 않는다).

## 결정 항목

이 컨벤션을 따르려면 다음 항목들을 결정하여 상세설계서에 명시해야 한다.

| 결정 항목 | 명시 위치 | 형태 / 제약 |
|----------|---------|-----------|
| 액션 강조 위계 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 1차/2차/3차(텍스트) 액션의 시각 구분 규칙. 한 화면/섹션의 1차 액션은 **1개** 원칙 |
| 정렬 그리드 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 콘텐츠 정렬 기준(예: 좌측 정렬 기본·단일 컬럼 그리드·최대 콘텐츠 폭). 가장자리 정렬 일관성 |
| 콘텐츠 밀도 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 셋 중 택일: ① 여유(넉넉한 여백) ② 보통 ③ 조밀(데이터 집약 화면). 섹션 간/그룹 간 간격 토큰 기준 |
| 강조 수단 우선순위 | `service/{프론트엔드 서비스}.md` / 디자인 정책 섹션 | 위계 표현에 쓰는 수단 순서(크기 → 굵기 → 색 → 간격). 색만으로 위계를 만들지 않는다(a11y) |

## 상세설계 작성 예

`service/{프론트엔드 서비스}.md` 에 다음 섹션을 추가한다.

```markdown
## 디자인 정책 - 시각 위계

| 항목 | 정책 |
|------|------|
| 액션 강조 위계 | 1차 = primary 채움 버튼(화면당 1개) · 2차 = outline/surface 버튼 · 3차 = 텍스트 링크 |
| 정렬 그리드 | 좌측 정렬 기본, 콘텐츠 최대 폭 1120px 중앙, 단일 컬럼 + 카드 그리드. 라벨/값 가장자리 정렬 |
| 콘텐츠 밀도 | 보통 — 섹션 간 `8`(32px), 그룹 간 `4`(16px), 항목 간 `2`(8px) |
| 강조 수단 우선순위 | 크기 → 굵기 → 색 순. 상태/위계를 색만으로 표현하지 않음(아이콘/텍스트 병행) |
```
