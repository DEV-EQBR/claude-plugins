# 블록체인 토큰 단위 처리

> 버전: v1.0.0
> 최종 수정: 2026-05-04
> 적용 조건: 다음 중 하나라도 해당하면 적용
> - 외부 시스템(`system.md`)에 블록체인 RPC 또는 토큰 컨트랙트 호출이 포함된 경우
> - 화면(`screens.md`)에 토큰 잔액/수량/송금액을 표시하는 요소가 있는 경우

---

## 메타-규칙

블록체인 토큰의 양은 **통신/저장용 raw 단위**(컨트랙트가 정의한 최소 정수 단위)와 **사용자 표시용 human 단위**(decimals 적용 후)를 분리한다. 변환은 시스템 내 단일 경계 위치에서만 수행한다. 토큰별 decimals 정보는 단일 출처에서 관리한다.

본 규칙은 네이티브 코인(ETH, SOL 등)과 토큰 표준(ERC-20, ERC-721, SPL 등) 모두에 적용된다.

---

## 결정 항목

이 컨벤션을 따르려면 다음 항목들을 결정하여 상세설계서에 명시해야 한다.

| 결정 항목 | 명시 위치 | 형태 / 제약 |
|----------|---------|-----------|
| 적용 토큰 목록 | `system.md` / 토큰 처리 섹션 | 토큰별 식별자(symbol/contract address/체인) + decimals 값 |
| 통신/저장 단위 정책 | `system.md` / 토큰 처리 섹션 | "raw 단위(decimals 적용 전 정수)로 통신·저장" 명시. 직렬화 형식(문자열/BigInt 등) 포함 |
| 표시 단위 정책 | `service/{프론트엔드 서비스}.md` / 표시 형식 섹션 | 토큰별 표시 단위 (보통 decimals 적용 후 human 형식). 표시 정밀도(소수 자릿수) 포함 |
| 변환 경계 위치 | `system.md` / 토큰 처리 섹션 | 둘 중 택일: ① 백엔드 응답 시점에 변환 후 응답 ② 백엔드는 raw로 응답, 프론트엔드 렌더링 직전 변환 |
| decimals 정보 출처 | `system.md` / 토큰 처리 섹션 | 셋 중 택일: ① 온체인 조회 ② 백엔드 메타데이터 ③ 프로젝트 상수(고정 토큰셋) |

> 표시 단위에서의 수 형식(천 단위 콤마, 큰 수 축약 등)은 본 컨벤션이 아닌 `service/frontend/number-display.md` 에서 별도 결정.

## 상세설계 작성 예

`system.md` 에 다음 섹션을 추가한다.

```markdown
## 토큰 처리

### 적용 토큰

| 토큰 | 식별자 | 체인 | decimals |
|------|-------|------|---------|
| DAI | 0x...컨트랙트 | Ethereum Mainnet | 18 |
| USDC | 0x...컨트랙트 | Ethereum Sepolia | 6 |

### 단위 정책

| 항목 | 정책 |
|------|------|
| 통신/저장 단위 | raw 단위(decimals 적용 전 정수). 직렬화는 문자열로 (정밀도 손실 방지) |
| 변환 경계 | 백엔드는 raw로 응답, 프론트엔드 렌더링 직전 변환 |
| decimals 출처 | 프로젝트 상수 테이블(고정 토큰셋) |
```

`service/{프론트엔드 서비스}.md` 에는:

```markdown
## 표시 형식 - 토큰

| 토큰 | 표시 단위 | 정밀도 |
|------|---------|-------|
| DAI | DAI (decimals 18 적용) | 소수점 둘째 자리 |
| USDC | USDC (decimals 6 적용) | 소수점 둘째 자리 |
```
