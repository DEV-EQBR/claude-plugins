# 모션 불변식 실측 패턴

모션은 시간축 위의 속성이라 정적 확인(소스 검사·단일 스크린샷)이나 프레임 눈 판독으로 판정할 수 없다. 앱이 출력한 **계측 프로브 로그**를 상세설계(`ui.md` 모션 섹션)의 불변식과 기계적으로 비교하여 이진 판정한다. 플랫폼 무관 — 출력 채널·기동 인자만 프로젝트 선언(system.md "검증 환경 컨벤션")에 따라 달라진다.

## 전제 (구현에 있어야 하는 것 — coder 의무)

| 장치 | 내용 |
|------|------|
| 모션 계측 프로브 | 검증 빌드에서 모션 관찰 사실을 구조화 로그 한 줄로 출력. **형식 고정**: `probe:<이벤트> key=value ...`. 사실만 출력, 판정 로직 없음 |
| 결정론적 UI 재생 모드 | 외부 의존(LLM·네트워크) 없이 고정 자극 시퀀스를 고정 간격으로 자동 재생하는 기동 수단 |

둘 중 하나라도 없으면 우회하지 않고 **구현 누락 결함**으로 보고한다 (도구 제한 아님 — coder 재구현 대상).

## 절차

### 1. 계측기 정합 확인 (소스)

프로브가 상세설계된 이벤트에 올바르게 붙어 있는지 소스로 확인한다. 잘못 심긴 프로브(예: 뷰 등장이 아니라 데이터 추가 시점에 로그)의 깨끗한 로그는 무의미하므로 실측 전에 확인한다. 프로브는 한 줄짜리라 이 감사는 짧다. *기능*을 소스 대조로 PASS하는 것은 금지지만, *계측기*의 정합 확인은 소스 대조가 맞는 용도다.

### 2. 결정론적 재생으로 구동

system.md 선언의 재생 모드(기동 인자 등)로 앱을 구동한다. 앱이 스스로 진행하므로 UI 자동화 드라이버가 필요 없다(호스트 입력 점유 없음). 재생 완료(선언된 종료 조건)까지 timeout을 걸고 대기한다.

### 3. 프로브 로그 수집 (비차단)

선언된 출력 채널에서 `probe:` 라인을 수집한다. 스트리밍 금지 — 재생 종료 후 비차단 조회.

### 4. 불변식 assert

수집한 라인을 파싱해 상세설계의 각 불변식을 판정한다. 판정은 이진(만족/위반)이며 위반은 수치로 집계한다.

| 불변식 형식 | 판정 방법 | 위반 보고 예 |
|------------|----------|-------------|
| **횟수** | 이벤트 카운트 — 신규 identity당 등장 이벤트 1회, 기존 identity 재등장 0회 | "id=t3 등장 2회 (재-전이 발생)" |
| **기하** | 좌표 비교 — 정착 시점 요소 경계 vs 가시 영역 경계 | "잘림 12pt (bottom 1212 > visible_bottom 1200)" |
| **시간** | 타임스탬프 차 — 자극 주입~정착 이벤트 vs 상세설계 상한 | "정착 480ms > 상한 300ms" |

## 프로브 로그 예 (리스트 삽입 모션)

```
probe:inject id=t3 t=1000            # 재생 모드가 자극 주입 (기준 시각)
probe:appear id=t3 bottom=1180 t=1120   # 신규 항목 뷰 등장
probe:settle visible_bottom=1200 t=1250 # 스크롤 정착 (기하 판정 시점)
```

- 횟수: `appear id=t3` 1회, 그 외 id의 `appear` 0회 → 만족
- 기하: 1180 ≤ 1200 → 만족 (잘림 0)
- 시간: 1250 − 1000 = 250ms ≤ 300ms → 만족

## 판정 제외

상세설계가 "사람 확인 항목"으로 분리한 감성 품질(easing 체감 등)은 판정하지 않고 보고에 그대로 전달한다. 스크린샷·프레임 추출은 판정 근거가 아니라 보조 증거로만 쓴다(시간·공간 해상도 부족).
